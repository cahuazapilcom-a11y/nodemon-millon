El backend es el cerebro de tu sistema. Aquí recibes los datos del usuario, construyes el prompt y le pasas la respuesta de la IA al frontend en tiempo real.

});

La Función de Streaming SSE

Esta es la función más importante de todo el sistema. Permite que el texto fluya hacia el usuario en tiempo real, exactamente como ChatGPT:

async function streamSSE(res, system, userContent, apiKey) {
  // Crear cliente con la API key del usuario
  const client = new Anthropic({
    apiKey: apiKey || process.env.ANTHROPIC_API_KEY
  });

  // Headers para Server-Sent Events
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  try {
    // Iniciar el stream con Claude
    const stream = client.messages.stream({
      model: 'claude-opus-4-6',
      max_tokens: 2500,
      system,
      messages: [{ role: 'user', content: userContent }],
    });

    // Enviar cada chunk de texto al cliente
    stream.on('text', (text) => {
      res.write(`data: ${JSON.stringify({ text })}\n\n`);
    });

    await stream.finalMessage();
    res.write('data: [DONE]\n\n'); // señal de fin

  } catch (err) {
    res.write(`data: ${JSON.stringify({ error: err.message })}\n\n`);
  } finally {
    res.end(); // siempre cerrar la conexión
  }
}

La Función de Lectura SSE en el Cliente

async function streamRequest(endpoint, body, outputEl) {
  outputEl.innerHTML = '';
  let fullText = '';

  const response = await fetch(endpoint, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-API-Key': state.apiKey // key del localStorage
    },
    body: JSON.stringify(body)
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop(); // último elemento puede estar incompleto

    for (const line of lines) {
      if (!line.startsWith('data: ')) continue;
      const raw = line.slice(6);
      if (raw === '[DONE]') return fullText;

      const { text } = JSON.parse(raw);
      if (text) {
        fullText += text;
        // Renderizar markdown en tiempo real
        outputEl.innerHTML = marked.parse(fullText);
      }
    }
  }
  return fullText;
}
⚡ Capítulo 07

Streaming en Tiempo Real — SSE

El streaming es lo que convierte una herramienta aburrida en una experiencia mágica. Ver el texto aparecer palabra por palabra genera engagement y reduce la tasa de abandono a casi cero.

¿Qué son Server-Sent Events?

SSE es un protocolo donde el servidor puede enviar datos al cliente continuamente a través de una sola conexión HTTP. Es perfecto para streaming de IA porque:

  • No necesitas WebSockets (más complejo)
  • Funciona nativo en todos los navegadores modernos
  • El cliente puede leer los datos en tiempo real
  • Si la conexión se corta, el navegador reconecta automáticamente

El Formato de los Eventos

Cada mensaje SSE tiene este formato exacto (los saltos de línea dobles son obligatorios):

# Formato estándar SSE
data: {"text": "Hola"}\n\n
data: {"text": " mundo"}\n\n
data: [DONE]\n\n

Manejo de Errores en el Stream

// En el cliente — siempre maneja errores del stream
try {
  const { text, error } = JSON.parse(raw);

  if (error) {
    outputEl.innerHTML = `
      <div class="error">
        ❌ Error: ${error}<br>
        Verifica que tu API key sea válida.
      </div>`;
    return '';
  }

  if (text) {
    fullText += text;
    outputEl.innerHTML = marked.parse(fullText);
  }
} catch (e) {
  // ignorar líneas mal formateadas
}
🎯
Truco de UX: Auto-scroll

Agrega outputEl.scrollIntoView({ behavior: 'smooth', block: 'end' }) cada vez que actualizas el innerHTML. El usuario verá el texto aparecer sin tener que hacer scroll manualmente — exactamente como ChatGPT.
